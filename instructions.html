Instructions

Before you do anything, make sure you have a local copy of the diagnostics project. You can either clone it from the GitHub repository, or you can just download a release. You will also need to know the location of your own project's TEI XML files on your computer.

How to run diagnostics on your project in Oxygen

To run diagnostics against a TEI XML encoding project, you will need:

  • A project folder which contains all of your TEI XML files. They can be nested in subfolders or organized in any way.
  • A single file of any kind in that root folder.
  1. In Oxygen, choose File / Open Project, and choose the diagnostics.xpr file to open the diagnostics project.
  2. Open this document in Oxygen.
  3. Press the red triangle button on the toolbar, or choose Document / Transformation / Apply Transformation Scenario(s).
  4. When prompted, choose a file in the root folder of your project directory.

You will see progress messages at the bottom of the Oxygen window, and eventually a diagnostics page will open in your web browser.

How to run diagnostics at the command line

  1. Make sure ant is installed on your computer.
  2. cd to the diagnostics project folder.
  3. Run this command:
    ant -lib utilities -Duser.input [path-to-your-project-folder]

A new folder will be created inside your project folder called diagnostics_[the-date], containing a file called diagnostics.html. Open this file in your browser to see the results of the diagnostics.
